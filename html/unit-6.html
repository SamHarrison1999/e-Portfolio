Encapsulation

Encapsulation is a concept in which related data and methods are grouped together, and in which access to data is restricted.

Public vs. Private

Public means that the attribute or method can be accessed by an instance of a class. Private means that the attribute or method can only be accessed by the class itself. In Python, all methods and attributes are public by default. Python acknowledges private values in a unique way it uses a double underscore for name mangling and a single underscore as a convention for naming private values.

Classes as encapsulation

Class are a form of encapsulation in python as they are used to group together data and methods.

Encapsulation through convention

Private Classes

In Python, when programmers use a single underscore before an attribute or method name, there is an informal agreement to recognize that attribute or method as private.

class Phone:
    def __init__(self, make, storage, megapixels, carrier):
        self._make = make
        self._storage = storage
        self._megapixels = megapixels
        self._carrier = carrier

>>> my_phone = Phone("iPhone", 256, 12, "AT&T")
>>> print(my_phone.__dict__)
{'_make': 'iPhone', '_storage': 256, '_megapixels': 12, '_carrier': 'AT&T'}

Double underscores

Using two underscores, causes the Python interpreter to enforce changes. It is used to avoid name collisions in inheritance by changing the name to the format _ClassName__AttributeName.

When the Python interpreter encounters an attribute with a double underscore, it changes the name to _ClassName__AttributeName. This process is called mangling.

class PrivateClass:
    def __init__(self):
        self.__private_attribute = "I am a private attribute"

    def helper_method(self):
        return self.__private_method()

    def __private_method(self):
        return "I am a private method"

obj = PrivateClass()
print(obj.helper_method())

Getters and setters are used to access and manipulate private attributes.

Getters with the Property Decorator

The property object in Python allows you to write code as if all the attributes are public. The property class is used to write getters and setters in a pythonic way. The property class can be implemented either with the property decorator or with the property function.

class Person:
    def __init__(self, name, age):
        self._name = name
        self._age = age

    @property
    def name(self):
        return self._name

    @property
    def age(self):
        return self._age

>>> c = Person("Calvin", 6)
>>> print(c.name)
Calvin
>>> print(c.age)
6

Setters with the Property Decorator

The setter decorator starts with a @ followed by the name of the getter method, and it ends with .setter

The parameters for a setter function are self and the new value.

class Person:
    def __init__(self, name, age):
        self._name = name
        self._age = age

    @property
    def name(self):
        return self._name

    @name.setter
    def name(self, new_name):
        if type(new_name) != str:
            raise TypeError("Names must be expressed as a string")
        self._name = new_name

    @property
    def age(self):
        return self._age

    @age.setter
    def age(self, new_age):
        if new_age < 0:
            raise ValueError("Age can not be less than 0")
        self._age = new_age

>>> c = Person("Calvin", "6")
>>> print(c.name)
Calvin
>>> print(c.age)
6
>>> c.age = 30
>>> c.name = "Peter"
>>> print(c.name)
Peter
>>> print(c.age)
30
>>> c.age = -17
ValueError: Age can not be less than 0
>>> c.name = False
TypeError: Names must be expressed as a string
print(c.name)
print(c.age)

Finish off the coffee journal class with the add coffee, save and show coffee methods.

# inside CoffeeJournal class
def add_coffee(self):
    self._new_coffee.append([self._roaster, self._country, self._region, self._stars])

def save(self):
    with open(self._file, 'a') as f:
        writer = csv.writer(f)
        writer.writerows(self._new_coffee)

def show_coffee(self):
    print()
    # if there is no information on any coffee, tell the user to add one
    if len(self._old_coffee) < 2 and len(self._new_coffee) == 0:
        print("Enter a coffee first")
    # if there is information in the CSV but not new coffee print the old coffee
    elif len(self._old_coffee) > 2 and len(self._new_coffee) == 0:
        for row in self._old_coffee:
            print(f"{row[0]:13} {row[1]:13} {row[2]:13}  {row[3]:13}")
    # print both the old coffee and the new coffee
    else:
        for row in self._old_coffee:
            print(f"{row[0]:13} {row[1]:13} {row[2]:13}  {row[3]:13}")
        for row in self._new_coffee:
            print(f"{row[0]:13} {row[1]:13} {row[2]:13}  {row[3]:13}")
    print()
