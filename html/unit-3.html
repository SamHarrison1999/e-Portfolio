E-portfolio Activities

Discuss which UML models are most applicable at different stages of the Software Development Life Cycle.

The different stages of the software development life cycle are; planning, analysis, design, implementation, testing and maintenance.

Use case diagrams should be created as soon as you begin your project as they help to identify common activities. Use cases are part of the requirements gathering process. Use case diagrams should be looked throughout the software development life cycle. For example, in testing a use case diagram helps you know what scenarios you need to test, and for analysis, a use case diagram can be used to write user stories which can help you figure out the goals of your project.

Class diagrams should be one of the first diagrams you create as they show the relationships between classes and the operations you can perform on each object. Class diagrams are helpful during the implementation phase of the software development life cycle as they detail the methods and attributes for each class. Not having any class diagrams can have a significant impact on a project as without a class diagram you could write extra unnecessary code. Class diagrams enable code reuse making your code cleaner.

Interaction diagrams show the interactions between objects because of this they should be used in the requirements gathering process and testing. The most common interaction diagram is a sequence diagram that shows the interactions with a system in sequential order.

Activity diagrams are used to show the flow of an application because if this they should be used during the requirements gathering process.
